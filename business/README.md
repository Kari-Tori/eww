# Business

## Zakres

- Modele domeny, opis procesu biznesowego i zasady produktowe.
- Wymagania, backlog wysokopoziomowy, SLA, analizy wpływu.
- Notatki warsztatowe i materiały dla interesariuszy nietechnicznych.

## Struktura

| Katalog | Opis |
| --- | --- |
| `business/governance/` | Strategia zarządzania repozytorium, cele, polityki, kontrakty repo. |
| `business/methodologies/` | Metodyki pracy (PARA, DevOps, Blitzscaling) oraz przewodniki. |
| `business/processes/` | Procedury biznesowe/BPMN, KPI i instrukcje operacyjne. |
| `business/vendors/` | Partnerzy, wykonawcy, SLA i kontakty. |
| `business/archive/` | Historyczne materiały zespołowe i projektowe. |

## Następne kroki

1. Przegląd dokumentów w `docs/` i katalogach użytkowników, które powinny trafić do warstwy biznesowej.
2. Uzgodnienie standardu nazw i struktury podkatalogów (np. `requirements/`, `processes/`, `stakeholders/`).
3. Dodanie indeksu odsyłającego do najważniejszych artefaktów biznesowych oraz integracja z `INDEX.md`.
